# backlog-infra — Backlog

A running list of work to do. Mark items `- [x]` when done and move
them to the Done section, `- [ ]` when open, `- [~]` when in progress,
`- [?]` when blocked on a design decision the user needs to weigh in
on, and `- [!]` once you've written answers inline and want the worker
to retry. The autonomous worker (`backlog run`) picks `- [ ]` and `- [!]` items and skips `- [?]`; surface those
to the user when waking them.

Items parked in `## Thinking` are never picked by the daemon — move
them to `## Open` when ready for work.

## Thinking

> **Rationalization (2026-05-28) — read before promoting items to `## Open`.**
> Each item is tagged **(priority · wave)**.
> **Priority:** P1 = prevents unattended damage / makes the fleet observable / foundational unblocker · P2 = output quality + correctness · P3 = larger architectural or meta-maintenance.
> **Wave (build order, derived from dependencies):** W0 foundations → W1 primitives → W2 consumers → W3 orchestration → W4 lifecycle.
>
> **Shared primitives — build once, then the tagged items are thin consumers (don't implement 5 bespoke versions):**
> 1. **Pause mechanism** (scope `item | project | fleet` → heartbeat-only, with reason + until-condition): account-cooldown, per-project budget, per-item ceiling, fleet-freeze, pause/resume.
> 2. **Tick guard + abort/reopen/annotate** (a check that can veto a tick): per-tick verification, circuit breaker, diff-size, secret-scan, protected-path, post-completion review.
> 3. **One notify/event channel** (extend `agent-notify.json`): dead-man's-switch, fleet digest, brittle-contract canary (+ existing cooldown alert).
> 4. **Structured event log** → feeds metrics, alerting, digest, monitor.
> 5. **D1 telemetry bus + liveness** (COMMITTED) → underpins the account-cooldown signal, fleet-freeze flag, version-skew, live dashboard. Sequence early as an enabler.
> 6. **Test/CI harness** → gates safely changing the driver for everything else. Do first.
> 7. **Backlog integrity**: schema lint + snapshot + the completion-flag recording item (in `## Open`).
>
> **Resolved decisions (2026-05-28):** D1 telemetry bus = **committed**. Compaction = **Archive (shared `backlog-agent-compact.mjs`) is canonical** — align per-project scripts. Branch-per-item isolation = **rejected**, in favor of commit-to-main + the damage guards (diff-size, verification, protected-path).

- [x] **(P1 · W1) `backlog-agents doctor` — preflight health command.** Added a read-only `doctor` subcommand to `bin/backlog-agents` that reports fleet drift in five groups: (1) manifest ↔ project-dir consistency (both directions), (2) launchd daemons — double-daemons (canonical + legacy label both loaded = hard fail), dangling plists, loaded-count, manifest projects with no daemon, (3) plists pointing at canonical de-symlinked paths (driver path matches the shared bin; WorkingDirectory resolves and isn't a symlink), (4) per-project config — required gitignore patterns + status-hook resolves (`node --check`), (5) globals & tools — budgets file present+valid, and `git`/`node`/`jq` (hard) + `gawk`/`fswatch` (warn — gawk is the BSD-awk inplace footgun). Live run already surfaced real drift (two projects missing gitignore patterns, gawk absent).
- [x] **(P1 · W0) Structured (JSON) event logs.** Added a machine-readable JSONL stream (`.claude/backlog-agent-events.jsonl`) alongside the freeform human log. The driver emits one JSON object per line for the key tick lifecycle events — `tick_start`, `claim` (+ `claim_lost`), `claude_exit`, `cooldown_armed`, `status_ok`/`status_fail`, and `tick_done` (with `outcome` + `duration_s`) — each with a common envelope (`ts`, `event`, `project`, `host`, `pid`). The emitter (`log_event` + a dependency-free `_json_escape`) is best-effort and never aborts a tick; the file is per-host, gitignored, and rotated at 10 MB. The freeform log is untouched (backward-compatible), so the existing monitor keeps working while downstream consumers (dead-man's-switch, efficiency metrics, fleet digest) can now match on a structured field instead of grepping prose. Covered by `test/events.bats` (6 tests). Migrating the monitor + building metrics off this stream are the W2 consumer items.
